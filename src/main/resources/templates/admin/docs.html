📦 Orders API

POST /v1/orders

Create a new order

# Create an order
curl -X POST http://localhost:9000/v1/orders \
  -u rzp_test_simulator:sim_secret_key_12345 \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10000,
    "currency": "INR",
    "receipt": "order_123"
  }'
GET /v1/orders/{order_id}

Fetch an order by ID

GET /v1/orders

Fetch all orders (supports pagination with skip & count)

💳 Payments API

GET /v1/payments/{payment_id}

Fetch a payment by ID

POST /v1/payments/{payment_id}/capture

Capture an authorized payment

# Capture a payment
curl -X POST http://localhost:9000/v1/payments/pay_xxx/capture \
  -u rzp_test_simulator:sim_secret_key_12345 \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10000
  }'
POST /v1/payments/{payment_id}/refund

Refund a payment (full or partial)

🔔 Webhooks

Supported Events

  • payment.authorized - Payment authorized (card blocked)
  • payment.captured - Payment captured (money received)
  • payment.failed - Payment failed
  • order.paid - Order fully paid
  • refund.created - Refund initiated
  • refund.processed - Refund completed
POST /v1/webhooks

Add a webhook configuration

# Configure webhook URL
curl -X POST http://localhost:9000/v1/webhooks \
  -H "Content-Type: application/json" \
  -d '{
    "url": "http://localhost:8089/webhooks/razorpay",
    "secret": "my_webhook_secret",
    "events": "*"
  }'

🛒 Checkout UI

Hosted Checkout

Redirect users to the checkout page:
http://localhost:9000/checkout/{order_id}

After payment, users are redirected to success/failure pages. The signature is provided for server-side verification. 

// Example: Redirect to checkout after creating order
const response = await fetch('http://localhost:9000/v1/orders', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ amount: 10000, currency: 'INR' })
});
const order = await response.json();

// Redirect to checkout
window.location.href = `http://localhost:9000/checkout/${order.id}`;
